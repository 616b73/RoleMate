# RoleMate Documentation

This directory contains the initial product and system documentation for RoleMate.

The goal of this documentation set is to give the project a clean foundation before development scales. It captures the product idea, MVP definition, user flow, system architecture direction, and phased delivery plan using common industry-style documentation.

## Documents

- `01-product-overview.md` - project summary, problem statement, value proposition, and vision
- `02-mvp-definition.md` - scoped MVP, non-goals, and success criteria
- `03-user-flow.md` - primary user journey and session lifecycle
- `04-system-design-draft.md` - high-level architecture and backend-first design notes
- `05-roadmap.md` - phased execution plan for building RoleMate

## Naming Note

This documentation replaces the old project name `MockMate` with `RoleMate` everywhere.

## Current Intent

Phase 1 is intentionally small:

- Role selection
- Queue-based matching
- Text chat
- Next partner action

Video, authentication, persistence, and advanced filtering are deferred until the core matchmaking loop is stable.
